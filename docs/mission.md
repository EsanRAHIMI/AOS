# Mission

Build a clean, reliable, governed AI operating system that first serves Esan as
primary owner, then scales to many users and roles without fragmenting the
software kernel. Personal data must be private and user-scoped; software
capabilities must remain global, coherent, and centrally governed.

## Current Mission

- Keep the 19-service kernel healthy, documented, and verifiable.
- Convert goals into traceable plans, agent runs, evidence, approvals, and reports.
- Use workspace isolation before any live code promotion.
- Preserve human control for risky or irreversible actions.
- Keep memory and documentation accurate enough that future agents can work cheaply.
- Treat Esan as the primary owner/admin while designing identity, data, and policy for more users.

## Next Mission

- Add user profiles, tenant boundaries, and personal/business/institutional memory layers.
- Connect read-only data sources first: calendar, email, files, tasks, projects, and approved public-service records.
- Add real web research with citations and freshness checks.
- Produce personalized daily briefings, weekly strategy reviews, and opportunity reports.
- Move from "kernel that can evolve itself" to "operating layer that improves each authorized user's day".

Success = an authorized user can state a goal, and AOS can plan, verify,
execute safe steps, ask for approval when needed, remember the result in that
user's scope, and recommend the next best move without confusion or invented
progress.

## Phase AA — platform governance direction
Esan is the first owner and platform governor. AOS is evolving into a
multi-tenant Autonomous Intelligence Government Platform serving future
personal users, teams, companies, government units, departments and citizens.
The invariant: **global software evolution, scoped human data** — one unified
kernel, hard isolation for every user/tenant/project/case boundary.
